## **2ï¸âƒ£ `axios`**
### **ðŸ”¹ What is `axios`?**
`axios` is a **promise-based HTTP client** for making API requests in JavaScript applications. It simplifies **sending HTTP requests** (GET, POST, PUT, DELETE, etc.) to APIs and handling responses.

### **ðŸ”¹ Why Use `axios`?**
âœ… **Handles API Requests Easily:** Simplifies calling external APIs from React.  
âœ… **Promise-Based:** Uses `async/await`, making requests easier to read and handle.  
âœ… **Supports Request Interceptors:** Modify requests before they are sent (e.g., adding authentication tokens).  
âœ… **Automatic JSON Parsing:** Converts API responses into usable JavaScript objects.  
âœ… **Handles Timeouts and Errors Gracefully:** Supports built-in error handling and retry mechanisms.  

### **ðŸ”¹ How to Install `axios`**
```sh

```

```sh

```

### **ðŸ”¹ How to Use `axios`**
#### **ðŸŸ¢ Basic GET Request**
```jsx

import { useEffect, useState } from "react";

const FetchData = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    axios.get("https://jsonplaceholder.typicode.com/posts/1")
      .then(response => setData(response.data))
      .catch(error => console.error("Error fetching data:", error));
  }, []);

  return (
    <div>
      <h2>Post Title: {data?.title}</h2>
    </div>
  );
};

export default FetchData;
```

#### **ðŸŸ  Making a POST Request**

```jsx
      userId: 1
    });

  } catch (error) {
    console.error("Error creating post:", error);
  }
};
```

#### **ðŸ”µ Setting Up an `axios` Instance for Authentication**
